üöÄ Getting Started

What is FortKnoxx?

FortKnoxx is an advanced security scanning platform that automatically checks your code for vulnerabilities, security issues, and quality problems. It combines 32 different scanning tools to give you comprehensive protection.

Why Use FortKnoxx?
  • Detects security vulnerabilities before they reach production
  • Finds hardcoded secrets and API keys in your code
  • Checks for outdated dependencies with known vulnerabilities
  • Identifies code quality issues
  • Uses AI to find zero-day vulnerabilities
  • Tests for business logic flaws
  • All scanners are free and open-source

How It Works

  1. Add Your Repository: Connect a GitHub/GitLab repository or upload code directly
  2. Configure Scanners: Choose which scanners you want to run (all enabled by default)
  3. Run Scan: Click "Scan Now" and FortKnoxx analyzes your entire codebase
  4. Review Results: See all vulnerabilities organized by severity
  5. Download Report: Get a professional PDF report you can share with your team

‚öôÔ∏è Configuring Settings

API Keys

Some scanners and AI features require API keys. Go to Settings ‚Üí API Keys to configure:

  • OpenAI API Key: For GPT-based security analysis (optional)
  • Anthropic API Key: For Claude-based analysis (optional)
  • Google Gemini API Key: For Gemini analysis (optional)
  • GitHub Token: For better rate limits and private repos
  • Snyk Token: For Snyk dependency scanning (optional, free tier available)
Note: API keys are encrypted before storage. Most scanners work without API keys - they're only needed for LLM-based security testing.

Scanner Settings

Go to Settings ‚Üí Scanners to enable or disable any of the 32 scanners. You can:

  • Turn off scanners you don't need
  • Disable resource-intensive scanners for faster scans
  • Enable DAST scanning when your app is running
  • Control which AI-powered features run
[Screenshot: Scanner Settings Page]
Tip: All scanners are enabled by default. Disabled scanners will be skipped during scans and won't show in results.

üîß Special Setup Instructions

Most scanners work out of the box! However, a few require extra setup:

üê≥ Docker (for ZAP DAST Scanner)

The OWASP ZAP DAST scanner runs in Docker to test live web applications.

Installation:

# macOS (using Homebrew) brew install --cask docker # Ubuntu/Debian sudo apt-get update sudo apt-get install docker.io # Windows Download from https://www.docker.com/products/docker-desktop

Verify Installation:

docker --version
How ZAP DAST Works:
  1. Your application must be running (e.g., http://localhost:3000)
  2. ZAP crawls the application and tests for vulnerabilities
  3. Detects: XSS, SQLi, security headers, SSL/TLS issues
  4. Takes longer than static scans (5-30 minutes)
Note: ZAP DAST is disabled by default. Enable it in Settings ‚Üí Scanners when you're ready to test a running application.

ü§ñ Training the Zero-Day Detector (GNN Model)

The Zero-Day Detector uses a Graph Neural Network that can be trained on your own data for better accuracy.

Quick Start (Use Pre-configured):

The model comes with pre-trained weights. Just enable "Zero-Day Detector" in scanner settings - no setup needed!

Advanced: Train Your Own Model

For best results, train the model on your codebase:

cd backend/engines/zero_day/training # Generate synthetic training data (25 vulnerability types) python3 generate_training_data.py --output ./data --samples 5000 # Train the GNN model (requires PyTorch) python3 train_model.py --data ./data --epochs 50 # Model will be saved to: ../checkpoints/deployment/
Requirements: Python 3.8+ PyTorch PyTorch Geometric 16GB RAM recommended

Training Tips:

  • First time? Use 1000-2000 samples and 20 epochs (faster)
  • Production use? Generate 10,000+ samples and 100+ epochs
  • GPU available? Training is 10x faster with CUDA
  • Training time: 30 minutes (CPU) to 5 minutes (GPU) for 5000 samples

‚ùì Frequently Asked Questions

General Questions

Q: Are all the scanners really free?

A: Yes! All 32 scanners are free and open-source. Some optional features (LLM Security Scanner) require API keys, but those are also optional.

Q: Can I scan private repositories?

A: Yes! Add a GitHub token with repo access in Settings ‚Üí API Keys. Your code never leaves your infrastructure.

Q: How long do scans take?

A: Typical scans take 5-10 minutes for medium codebases (10K-50K lines). Large codebases or enabling all scanners may take 20-30 minutes.

Q: Do I need all 32 scanners enabled?

A: No! You can disable scanners you don't need. For faster scans, disable quality scanners and keep security scanners enabled.

Technical Questions

Q: Does FortKnoxx support my programming language?

A: FortKnoxx supports 20+ languages including:

  • Python: ‚úÖ Excellent coverage (8 dedicated scanners)
  • JavaScript/TypeScript: ‚úÖ Excellent coverage
  • Java: ‚úÖ Good coverage
  • Go: ‚úÖ Good coverage
  • Ruby, PHP, C/C++: ‚úÖ Supported
  • Rust, Kotlin, Swift: ‚ö†Ô∏è Partial support

Q: What's the difference between SAST and DAST?

A:

  • SAST (Static): Analyzes source code without running it. Fast, finds more issues. All scanners except ZAP DAST are SAST.
  • DAST (Dynamic): Tests running applications. Finds runtime issues and real-world exploitability. Only ZAP DAST does this.

Q: Why do I see duplicate findings?

A: Multiple scanners may report the same issue. This actually increases confidence! FortKnoxx shows which scanners found each issue.

Q: Can I automate scans in CI/CD?

A: Yes! FortKnoxx has a REST API. You can trigger scans automatically in GitHub Actions, GitLab CI, or any CI/CD pipeline.

AI Scanner Questions

Q: What is the Zero-Day Detector?

A: It's a Graph Neural Network (GNN) trained to find vulnerability patterns that traditional scanners miss. It learns from code structure and data flow to detect anomalies.

Q: Do I need to train the model?

A: No! It comes pre-trained. Training your own model can improve accuracy for your specific codebase, but it's optional.

Q: How accurate is the Zero-Day Detector?

A: It has ~75-85% precision on common vulnerability types. Expect some false positives - always manually review flagged code.

Q: What's the Business Logic Scanner?

A: It analyzes application flow to find business logic flaws like:

  • IDOR (Insecure Direct Object References)
  • BOLA (Broken Object Level Authorization)
  • Race conditions
  • Privilege escalation paths

Troubleshooting

Q: A scan failed. What should I do?

A: Check:

  1. Is the repository accessible?
  2. Is there enough disk space? (5GB+ recommended)
  3. Are all required tools installed? Run: ./install_all_scanners.sh
  4. Check logs for specific scanner errors

Q: ZAP DAST is not working

A: Verify:

  • Docker is installed and running: docker ps
  • Your application is running and accessible
  • ZAP DAST is enabled in scanner settings

Q: The Zero-Day Detector shows an error

A: Make sure:

  • PyTorch is installed: pip3 install torch
  • PyTorch Geometric is installed: pip3 install torch-geometric
  • Model checkpoint exists: backend/engines/zero_day/checkpoints/deployment/

Q: Some scanners show "not available"

A: Run the installation script:

cd /path/to/fortknox ./install_all_scanners.sh
