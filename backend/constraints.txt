# pip constraints file — enforces ceilings that prevent known conflicts.
#
# WHY a separate constraints file instead of hard pins in requirements:
#   - Constraints are applied on top of requirements without changing what
#     packages are installed (they only limit versions pip may choose).
#   - This lets transitive deps resolve freely while hard-blocking known-bad
#     versions.
#
# Usage:
#   pip install -r requirements.txt -c constraints.txt
#
# Conflict map (as of 2026-05):
#   checkov 3.x          → networkx<2.7, cachetools<6, importlib-metadata<8,
#                           packaging<24
#   torch 2.x            → setuptools<82
#   transformers 4.x     → stay below 5.0 (regex + tokenizers incompatible)
#   safety 3.7+          → filelock>=3.16.1, typer>=0.16
#   huggingface-hub 1.13 → hf-xet>=1.4.3
#   pip-audit 2.10+      → tomli>=2.2.1
#   cyclonedx-bom <7     → cyclonedx-python-lib<8
